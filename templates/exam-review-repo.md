# 应试复习仓库建设模板

这个模板用于把一门课程建设成“可被 AI 持续辅助复习”的仓库。建设目标是应试，不是百科式学习。

## 1. 目录建议

推荐结构：

```text
课程名/
  AGENTS.md
  CLAUDE.md
  files-from-teacher/
    TD/
    cours/
  agents/
    plan.md
  notes/
  scripts/
```

如果原仓库已经有目录约定，优先沿用原约定。

## 2. 资料盘点

先盘点 `files-from-teacher/` 下所有资料：

- PDF、PPT、Word、Excel、图片、视频、音频。
- 哪些是题目来源：TD、作业、correction、历年题、老师讲题视频。
- 哪些是课程讲义：cours、slides、lecture notes。
- 哪些是教材或参考书。
- 哪些资料不适合大模型直接阅读。

不确定文件含义时必须向用户提问。

## 3. 应试优先级写法

不要机械写成“A -> B -> C”就结束。应该在提示词中解释为什么这样排：

```markdown
资料使用必须服从应试目标：

1. TD、correction、老师讲题视频是最高优先级，因为它们最接近题目来源。判断考试重点、题型、答题方式和常见陷阱时，必须先看这些资料。
2. 老师课件用于补充定义、机制、公式来源和例子。
3. 参考书只作为弱优先级背景资料。只有当 TD 和课件不足以解释基础概念时才使用，不要让参考书反过来主导复习路线。
```

## 4. 课程级提示词必须包含

`AGENTS.md` 和 `CLAUDE.md` 建议保持一致，至少包含：

- 项目目标：从知识点维度逐点复习，用户默认零基础。
- Teach 技能默认使用：复习教学时直接使用 Teach 技能提高讲解质量，但不改变本课程的资料优先级、考试范围、答题语言和来源标注。
- 资料范围与优先级：题目来源最高，课件补充，参考书弱优先级。
- 语言要求：讲解中文；考试答案按课程要求语言；必要时附中文翻译。
- 题目讲解格式：题目、解析、答案。
- 知识点讲解格式：前置概念、核心定义、为什么重要、考试问法、答题模板、易错点、小练习。
- 来源标注格式。
- 资料可读化规则。
- 图形知识点生成 HTML 的规则。
- 不确定情况必须问用户。

Teach 技能默认使用建议写法：

```markdown
## Teach 技能默认使用

帮助用户复习时，必须直接使用 Teach 技能提高教学质量和复习效率。Teach 是教学方法要求，不替代本课程的应试资料优先级、考试范围、答题语言、来源标注和题目讲解格式。

默认用户是小白。讲知识点、讲题、讲公式、讲图形或讲答题模板时，先补必要前置概念，每次只推进一个最小知识点或一道题中的一个关键步骤，不用“显然”“易得”“可知”等跳步表达代替解释。

必须把知识点连接到应试题源：说明它为什么重要、可能怎么考、常见陷阱是什么。给出考试可写的答题模板，但不能只给模板而不解释。通过一个小练习、追问或自测点检查用户是否真的理解；用户答错时，回退到缺失的前置概念继续教。

只有用户明确说“只要答案”“只要模板”“不要展开”时，才压缩教学过程。
```

## 5. 题目讲解格式模板

```markdown
## 题目

原题：
> [保留题目原语言]

中文翻译：
> [中文翻译]

## 解析

[中文讲前置概念、思路、公式、易错点]

## 答案

考试版：
> [用课程要求语言写完整答案]

中文翻译：
> [对应中文翻译]
```

如果课程考试要求法语，考试版就必须用法语。

## 6. 资料可读化规则

转换时不覆盖原文件，输出放在原文件同一目录。

推荐命名：

- PDF 可读版：`原文件名.readable.txt`
- OCR 版：`原文件名.ocr.txt`
- PDF 中文翻译：`原文件名_中文翻译.md`
- 视频转写：`原文件名.transcript.<lang>.md`
- 视频转写中文翻译：`原文件名_音频转写_中文翻译.md`

视频 ASR 要指定课程语言。例如法语课程使用 `language=fr`。

大文件或参考书处理原则：

- 如果参考书很大，先问用户。
- 用户确认跳过时，不做 OCR、不翻译，只在提示词中标为弱优先级。
- 用户确认处理时，再按章节或分块处理，避免一次生成巨大低价值产物。

## 7. 复习计划要求

`agents/plan.md` 必须从老师资料出发，尤其从题目来源反推知识路线。

必须避免时间维度噪音：

- 不写第几天。
- 不写周计划。
- 不写每日打卡。

推荐结构：

- Summary
- Source Map
- Knowledge Route
- High-Priority Exam Patterns
- Review Method
- Assumptions

## 8. 可视化建设

遇到图形或过程类知识点，优先生成 HTML：

- 网络：分层、拓扑、路由、子网、帧格式、滑动窗口、波形编码。
- 数学：函数图、迭代过程、误差变化。
- 质量管理：控制图、抽样曲线、鱼骨图、Pareto、流程图。

HTML 必须注明来源文件和题号/页码。

## 9. 验收清单

完成一个课程复习仓库建设后，至少检查：

- `AGENTS.md` 和 `CLAUDE.md` 是否一致。
- 题目来源优先级是否明确。
- 是否说明参考书弱优先级。
- 是否存在不该提交的 API key 或密钥。
- 是否没有临时文件残留。
- 可读化产物是否覆盖了用户确认要处理的资料。
- `agents/plan.md` 是否按知识点组织，且没有时间安排噪音。
